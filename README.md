# Image Rotator

![Image Rotator — превью окна](docs/screenshot.jpg)

Маленькое десктопное приложение под Windows для пакетного поворота PNG-изображений с сохранением прозрачности.

Бросаете в окно файлы или целую папку, поворачиваете каждое изображение перетаскиваемой ручкой (со снапом к 0/90/180/270), а на выходе получаете обработанные PNG (с обрезкой прозрачных краёв) в любой папке. Если рядом с PNG лежит `sizes.json` от `nakley-menya-export`, приложение подхватит физические размеры стикеров и пересчитает их с учётом поворота и обрезки.

## Что нового в 1.2.0

- **Новый фирменный логотип Rotate Arc** — рамка изображения, обёрнутая круговой стрелкой поворота. Знак отражает суть приложения и заменил прежнюю заимствованную иконку. Внедрён в иконку приложения (`.exe`) и установщика, favicon и шапку интерфейса; фирменный цвет — бирюза `#23d9b7`.

## Что нового в 1.1.0

- **Поддержка `sizes.json`** — приложение принимает JSON-метаданные стикеров от проекта `nakley-menya-export` тремя способами: автоматически при перетаскивании папки, drag&drop отдельного `.json`, или отдельной кнопкой «Загрузить sizes.json».
- **Физические размеры на карточках** — под каждым PNG показывается размер стикера в миллиметрах (с белыми полями). При повороте отображается прогноз итогового размера.
- **Обновлённый `sizes.json` на выходе** — при экспорте рядом с PNG создаётся новый `sizes.json` с точно пересчитанным `png_size_mm` для каждого повёрнутого/обрезанного стикера. Добавлено поле `angle_deg`. PNG без записи в JSON в выходной файл не попадают.
- **Статус-индикатор в Toolbar** — показывает, сколько PNG сопоставлено с записями из JSON, кнопка экспорта блокируется, пока в фоне идёт чтение pixel-размеров.

Готовые сборки лежат в [GitHub Releases](https://github.com/baslie/image-rotator/releases/latest):

- **`image-rotator.exe`** — портативный standalone, ~3.5 MB, запускается без установки. На Windows 11 WebView2 уже встроен.

## Как работает связка с `sizes.json`

1. Дизайнер выгружает стикеры из Illustrator через [`nakley-menya-export`](https://github.com/baslie/nakley-menya-export) — получает папку с `sticker_001.png`, `sticker_002.png`, … и `sizes.json`.
2. Перетаскивает папку в Image Rotator. Приложение автоматически находит `sizes.json` и привязывает к каждому PNG исходные физические размеры (cut + белая обводка + прозрачные поля).
3. Дизайнер визуально выравнивает повёрнутые стикеры (нейронкам нужно «эталонное» изображение по центру). На карточке сразу видно, какой размер в мм получится после поворота и обрезки.
4. На экспорте все PNG проходят rotate → trim прозрачных краёв, а в той же папке создаётся обновлённый `sizes.json`, где `png_size_mm` точно соответствует итоговому файлу. Этот JSON дальше используется при сборке промпта для Nano Banana Pro и других моделей.

Точная математика пересчёта: исходный PNG имеет известное `png_size_mm` (в мм) и pixel-размер. Из них считается `mm_per_px`. После rotate+trim получаем финальный pixel-размер; умножаем на `mm_per_px` → точный физический размер итогового PNG в мм. Поскольку `trim` удаляет только прозрачные пиксели (белая обводка остаётся), итог совпадает с реальным bbox повёрнутого стикера с белыми полями.

## Стек

- **Tauri 2** — нативная Windows-оболочка (Rust + WebView2)
- **React 19 + TypeScript + Vite 8** — фронтенд
- **Tailwind v4 + shadcn/ui + Zustand** — UI и состояние
- **NSIS** — инсталлятор

## Сборка из исходников

Один раз поставить:

1. [MSVC C++ Build Tools](https://aka.ms/vs/17/release/vs_BuildTools.exe) — workload «Desktop development with C++» (MSVC v143 + Windows 11 SDK)
2. [Rust toolchain](https://rustup.rs)
3. Node.js 20+

Затем из `app/`:

```powershell
npm install
npm run tauri:dev      # окно с HMR
npm run tauri:build    # production-инсталлятор → app/src-tauri/target/release/bundle/nsis/
```

## Структура

- `app/` — фронтенд (React + Vite)
- `app/src-tauri/` — Rust-крейт `image-rotator`, `tauri.conf.json`, иконки, capabilities
- `docs/` — заметки по интеграции (`nano-banana-pro-razmery.md` — про работу нейросетей с физическими размерами)
